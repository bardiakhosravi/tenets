> Installed by [tenets](https://github.com/bardiakhosravi/tenets). Run `npx tenets update` to update.

## How to use these rules

These rules define how to build Python backend services using Domain-Driven Design and Hexagonal Architecture (Ports & Adapters). When writing or reviewing code in this project:

1. **Follow every rule below.** They are organized into four sections: Architecture, Domain, Application, and Global.
2. **Architecture** defines the structural patterns — ports, adapters, layers, and how they connect.
3. **Domain** defines how to model business logic — entities, value objects, aggregates, events, and bounded contexts.
4. **Application** defines how to orchestrate business operations — use cases, port synergy, and cross-context communication.
5. **Global** defines cross-cutting standards — project structure, testing, naming, dependencies, validation, and error handling.
